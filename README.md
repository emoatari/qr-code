## What I learned

### Challenges I Faced:
- **Centering the card:** I initially used `padding: 300px` which broke on mobile. I learned that using `display: flex` with `min-height: 100vh` is a much more reliable and responsive way to center elements.
- **Commit messages:** My first messages were vague ("update readme"). I understood that writing clear, imperative messages like `fix: correct padding` makes the project history more useful for myself and others.

### Key Takeaways:
- **CSS Flexbox:** I practiced using `flexbox` for layout and realized how powerful it is for alignment.
- **Accessibility (a11y):** I learned the importance of `alt` text for images and using a proper heading hierarchy (`<h1>` to `<h6>`).
- **Performance:** Using `preconnect` for Google Fonts is a small but impactful optimization I will use in future projects.

### Next Steps:
- I will apply these learnings to my next project and focus on writing better commit messages from the start.
